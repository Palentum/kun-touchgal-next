# VNDB Tag 同步策略

## 背景

网站的 `patch_tag` 原先全部来源于 bangumi。为丰富标签数据，新增从 VNDB 自动获取标签的功能。

## 数据模型变更

在 `patch_tag` model 新增字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `source` | String, 默认 "self" | 标签来源: self（用户创建）、bangumi、vndb |
| `vndb_original_name` | String? | VNDB tag 英文原名，用于去重 |
| `vndb_category` | String? | VNDB 分类: cont / tech |
| `vndb_spoiler` | Int? | 剧透级别: 0/1/2 |
| `vndb_rating` | Float? | 相关度评分: 0~3 |

## Tag 获取过滤策略

### 分类过滤

VNDB tag 有三个分类：
- `cont` (Content) — 内容标签，如 Romance、Mystery、Fantasy
- `ero` (Sexual) — 色情内容标签
- `tech` (Technical) — 技术标签，如 ADV、Multiple Endings、Voice Acting

**决策：仅获取 `cont` 和 `tech` 分类的标签**，排除 `ero` 分类。

### 剧透过滤

VNDB tag 的剧透级别：
- `0` — 无剧透
- `1` — 轻微剧透
- `2` — 严重剧透

**决策：仅获取 `spoiler === 0`（无剧透）的标签**，避免在标签中泄露剧情。

### 其他过滤

- `lie === true` 的标签被排除（VNDB 标记为错误应用的标签）
- `rating` 字段仅记录到数据库，不作为过滤条件

## 名称映射

VNDB tag 为英文名。通过 `lib/tagMap.ts` 中的 `TAG_MAP` 映射为中文名作为 `patch_tag.name`，英文原名存储在 `vndb_original_name` 字段。

未命中映射表的 tag 保留英文原名。

## 去重策略

1. **通过 `vndb_original_name` 去重**：优先匹配已有的 VNDB 来源标签
2. **通过映射后的中文 `name` 去重**：避免与已有标签（bangumi/self 来源）名称冲突
3. 已存在的标签直接建立关联，不重复创建

## 触发时机

- 创建 Galgame 时，若提供了 VNDB ID，在用户 tag 处理后自动获取 VNDB tag
- 更新 Galgame 时，同上
- 批量同步脚本：`migration/syncVndbTags.ts`

## VNDB API 限速

- VNDB API 限制: 200 requests / 5 minutes
- 批量同步脚本每 50 个请求后等待 2 秒
- 使用反代 API `vndbapi.arnebiae.com` 避免直连超时
