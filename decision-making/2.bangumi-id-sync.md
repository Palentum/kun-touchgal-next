# Bangumi ID 同步策略

## 背景

为 patch 新增 `bangumi_id` 字段，关联 Bangumi 条目，为后续获取 Bangumi 标签等数据做准备。

## 数据模型变更

在 `patch` model 新增字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `bangumi_id` | Int?, unique | Bangumi 条目 ID |

## 搜索匹配策略

### API 选择

使用 Bangumi v0 实验性搜索 API：`POST /v0/search/subjects`

- 筛选条件：`type: [4]`（游戏），`nsfw: true`（包含 NSFW 结果）
- 排序：`match`（按匹配度）
- 每次返回最多 10 条结果

### 搜索顺序

1. 先用 patch 的 `name` 搜索
2. 如果未匹配到，再逐个用 `patch_alias` 的 `name` 搜索
3. 找到匹配即停止

### 匹配规则

**严格精确匹配**：搜索结果中的 `name`（日文名）或 `name_cn`（中文名）必须与 patch 的 `name` 或任一 `alias` 名称完全相同（字符串全等），才视为匹配成功。

不做模糊匹配，避免错误关联。

### 去重规则

- `bangumi_id` 设置了 `@unique` 约束
- 同步时维护已占用 ID 集合，如果某个 `bangumi_id` 已被其他 patch 占用，则跳过当前 patch

## 限速策略

- 每次 API 请求间隔 500ms
- 遇到 HTTP 429 响应时，自动等待 5 秒后重试

## API 认证

- 使用 Bearer Token 认证：`Authorization: Bearer ${KUN_BANGUMI_TOKEN}`
- Token 存储在 `.env` 文件的 `KUN_BANGUMI_TOKEN` 变量中
- User-Agent 格式遵循 Bangumi 要求：`touchgal/1.0 (https://www.touchgal.top)`
