# Bangumi Tag 获取策略

## 背景

在已有 VNDB tag 获取的基础上，新增从 Bangumi 获取标签的功能。Bangumi 标签为中文，无需映射，可直接使用。

## API 选择

使用 Bangumi v0 API：`GET /v0/subjects/{id}`

- 认证方式：Bearer Token（`KUN_BANGUMI_TOKEN`）
- User-Agent：遵循 Bangumi 要求，格式为 `touchgal/1.0 (https://www.touchgal.top)`
- 返回数据包含 `tags` 数组，每个 tag 有 `name` 和 `count` 字段

## Tag 过滤策略

### 热度过滤

仅导入 `count >= 10` 的标签。低于此阈值的标签通常为个人标注，不具代表性。

### 低质量标签过滤

使用 `lib/bgmDirtyTag.ts` 中维护的 `lowQualityTags` 列表过滤低质量标签，包括：

- **平台标签**：PC、PS4、PS5、NS、Android、iOS 等
- **状态标签**：通关、想玩、已购、全CG 等
- **翻译相关**：汉化、机翻、AI翻译、GPT翻译 等
- **主观评价**：神作、庸作、垃圾 等
- **通用分类**：Galgame、GAL、游戏、18+ 等
- **其他噪音**：个人备注、不规范标签等

该列表集中维护，便于后续扩展。

## 去重策略

- 通过标签 `name` 直接匹配：Bangumi 标签为中文，与数据库中已有标签名直接比较
- 已存在同名标签时，仅建立关联，不重复创建
- 使用 `skipDuplicates` 防止关联记录冲突

## 跳过条件

如果 patch 已有 `source: 'bangumi'` 的标签关联，直接跳过，不重复获取。

## 触发时机

- 创建 Galgame 时，若提供了 Bangumi ID
- 更新 Galgame 时，若提供了 Bangumi ID
- 均在用户 tag 处理和 VNDB tag 获取之后执行

## 与其他来源 Tag 的关系

三个外部来源的标签独立获取、独立存储：

| 来源    | source 值 | 名称语言  | 映射方式        | 过滤列表                             |
| ------- | --------- | --------- | --------------- | ------------------------------------ |
| VNDB    | `vndb`    | 英文→中文 | `lib/tagMap.ts` | 无（通过 spoiler/lie/category 过滤） |
| Bangumi | `bangumi` | 中文      | 无需映射        | `lib/bgmDirtyTag.ts`                 |
| Steam   | `steam`   | 中文      | 无需映射        | `lib/steamDirtyTag.ts`               |
| 用户    | `self`    | 中文      | —               | —                                    |

若不同来源产生同名标签，只会创建一条记录，多个 patch 关联到同一标签。
