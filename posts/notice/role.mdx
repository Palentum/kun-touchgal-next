## 角色对应关系

| Role | 角色名     | 说明                           |
| ---- | ---------- | ------------------------------ |
| 1    | 用户       | 默认注册用户                   |
| 2    | 创作者     | 可以使用对象存储上传资源的用户 |
| 3    | 管理员     | 负责站点内容维护               |
| 4    | 超级管理员 | 拥有最高后台权限的管理员       |

## 各角色详细说明

### Role 1 用户

| 项目     | 说明                                           |
| -------- | ---------------------------------------------- |
| 基础互动 | 可以关注、收藏、评论、评分、发送消息、提交反馈 |
| 资源发布 | 可以发布资源，但只能使用普通用户可用的资源方式 |
| 上传限制 | 不能上传到 `S3`，不能使用 `TouchGal` 资源盘    |
| 审核机制 | 首个资源需要审核，通过后才会公开显示           |
| 成长路径 | 累计发布至少 3 个资源后，可以申请成为创作者    |

### Role 2 创作者

| 项目     | 说明                                                                |
| -------- | ------------------------------------------------------------------- |
| 继承能力 | 拥有普通用户全部功能                                                |
| 新增权限 | 可以上传文件到 `S3` 对象存储                                        |
| 上传条件 | 需要萌萌点大于等于 20，且受每日 5GB 上传额度限制                    |
| 额外校验 | 上传文件时需要完成人机验证                                          |
| 权限边界 | 仍然不能管理 Galgame 本体、标签、会社，也不能使用 `TouchGal` 资源盘 |

### Role 3 管理员

| 项目     | 说明                                                                 |
| -------- | -------------------------------------------------------------------- |
| 内容管理 | 可以编辑已有 Galgame 本体，但不能创建或删除 Galgame                  |
| 游戏维护 | 可以修改 Banner、管理标签、会社、游戏关联会社，并可从 VNDB 拉取会社 |
| 资源管理 | 可以发布资源，但不能使用 `TouchGal` 资源盘                           |
| 越权处理 | 可以编辑或删除其他用户的评论、评分、资源                             |
| 后台权限 | 不可访问 `/admin` 路径下的后台接口与页面                             |
| 权限边界 | 不可处理创作者申请，不可编辑其他用户信息                             |

### Role 4 超级管理员

| 项目       | 说明                                     |
| ---------- | ---------------------------------------- |
| 继承能力   | 拥有管理员全部权限                       |
| 用户管理   | 可以删除用户                             |
| 管理员管理 | 可以将其他用户设置为管理员               |
| 安全管理   | 可以强制关闭其他用户的两步验证           |
| 全站操作   | 可以向全站启用邮件通知的用户发送群发邮件 |
